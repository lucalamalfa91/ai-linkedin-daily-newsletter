#1 Latent Space ★ 10/10

Giving Agents Computers — Ivan Burazin, Daytona

May 21, 2026

↳ WHAT'S NEW Daytona, the developer sandbox infrastructure company, is running 850K daily sandbox executions and reporting 74% month-over-month growth. CEO Ivan Burazin walked through their architecture on the Latent Space podcast — bare metal sandboxes, RL-based evals, and what they're calling "Agent Cloud." ↳ THE REAL STORY The core problem Daytona is solving: agents need to run code somewhere, and that somewhere needs to spin up fast, stay isolated, and not cost a fortune at scale. Bare metal sandboxes (as opposed to containerized VMs on shared cloud) means lower latency and more predictable performance — relevant when an agent is doing iterative code execution in a loop and every cold start adds up. The 850K daily runs number is the real signal here: it tells you how much agent-driven code execution is already happening in production, not in demos. The RL evals angle suggests they're also thinking about sandboxes as the environment layer for training and evaluating coding agents — not just serving them.

Claude's take

74% MoM growth is a striking number, but it compounds from whatever baseline Daytona started at — and developer infrastructure companies are notorious for measuring 'runs' in ways that inflate activity (retries, health checks, internal tests all count). The more interesting open question is margin: bare metal is faster but operationally heavier than renting Lambda or Firecracker VMs. If the agent compute wave materializes at the scale everyone is projecting, Daytona's infrastructure bet pays off. If agent workflows consolidate inside the big cloud providers' own sandbox offerings — AWS already has Code Interpreter-style execution, Google has its own sandbox layer — Daytona's independence becomes a liability rather than a differentiator.

#2 Latent Space ★ 9/10

[AINews] All Model Labs are now Agent Labs

May 23, 2026

↳ WHAT'S NEW Latent Space flags a structural shift across the AI industry: model labs — OpenAI, Anthropic, Google DeepMind, and others — are repositioning themselves as agent labs. The framing isn't about a single product drop but a pattern visible across recent announcements, hiring, and API surface area. ↳ THE REAL STORY The practical implication is that the competitive axis is moving. Winning on benchmark scores for a base model matters less when the labs are now racing to own the full agent stack: memory, tool use, orchestration, and long-horizon task execution. For developers building today, this means the APIs you depend on are about to get more opinionated — labs will ship their own memory layers, their own tool registries, their own agent loops. The question shifts from 'which model is smartest?' to 'whose scaffolding do I want to be locked into?'

Claude's take

The catch is that agent infrastructure built by model labs has an obvious conflict of interest: it will be optimized to keep you calling their models, not to give you clean abstractions you can swap out. Every convenience feature — managed memory, built-in tool calling, hosted agent runners — is also a retention mechanism. Developers who adopt these stacks early will move fast, but the switching costs six months from now could be significant. Worth watching whether any open-source orchestration layer (LangGraph, smolagents, or something not yet named) gains enough traction to act as a neutral substrate before the proprietary stacks calcify.

↳ WHAT'S NEW Datasette Agent is the first release of an AI assistant built directly into Datasette, combining Simon Willison's LLM Python library with the Datasette data exploration tool after three years of parallel development. ↳ THE REAL STORY For developers already using Datasette to publish and explore SQLite databases, this means natural language querying and data interaction without wiring up a separate LLM integration. The LLM library handles model access and tool calling; Datasette provides the data layer and UI. The 'extensible' framing suggests plugin authors can add their own tools on top of the agent, which fits Datasette's existing plugin ecosystem.

Claude's take

Datasette's user base skews toward journalists, researchers, and solo developers — people who often work with sensitive or unpublished data. How the agent handles data privacy (local models vs. API calls to third-party providers) will determine whether this is actually usable for the most common Datasette workloads, or just a demo feature for public datasets.

#4 SQLite ★ 8/10

sqlite AGENTS.md

May 27, 2026

↳ WHAT'S NEW SQLite added an AGENTS.md file to its repository — a plain-text guide not for SQLite's own development workflow, but for developers pointing AI coding agents at the SQLite codebase. ↳ THE REAL STORY This is infrastructure for the agentic layer: a README-style contract that tells an autonomous coding agent how to navigate one of the most consequential C codebases in existence. SQLite ships in every iOS device, Android phone, browser, and Python installation on the planet — so an agent making uninformed edits here has a blast radius most repos don't. AGENTS.md is the emerging convention (Codex, Claude, and similar tools look for it by default) for giving agents repo-specific rules: what to build, how to test, what never to touch. SQLite having one signals that maintainers are now actively thinking about agent-assisted contributions from the outside, not just human PRs.

Claude's take

↳ WORTH WATCHING The interesting question isn't that SQLite added this file — it's what they put in it. The excerpt cuts off before the actual content, which is where the signal lives. Constraints on the amalgamation build? Warnings about the custom Lemon parser? Rules around the strict no-third-party-dependency policy? Whatever SQLite's maintainers felt necessary to spell out for an agent is a precise map of where autonomous coding tools are most likely to go wrong in a mature, idiosyncratic C codebase. Worth reading the actual file, not just the headline.

May 28, 2026

↳ WHAT'S NEW Cognition raised $1B at a $26B valuation in a Series D round — making it one of the most aggressively valued AI coding companies on the market right now. ↳ THE REAL STORY The framing here is deliberate: coding as an "uncapped TAM" is the investment thesis, not just a tagline. At $26B, Cognition is being valued not on current revenue but on the premise that AI agents will eventually absorb a meaningful share of all software development work — a market that compounds as software itself expands. For developers building with or alongside tools like Devin, this signals that the agentic coding space is attracting capital at a scale that will accelerate product development, hiring, and likely aggressive pricing strategies to capture enterprise contracts fast.

Claude's take

↳ WORTH WATCHING The gap between valuation and demonstrated traction is worth scrutinizing. Cognition's Devin generated significant buzz at launch but also significant pushback from practitioners who found real-world performance underwhelming relative to the demo. A $26B valuation requires either a rapid capability jump or a very long investor time horizon — and $1B in fresh capital creates pressure to show enterprise revenue quickly. Watch whether the next product cycle closes the gap between benchmark performance and what developers actually ship with it, or whether this becomes a cautionary tale about AI coding hype outpacing utility.
